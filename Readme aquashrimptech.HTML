Sistema de monitoramento e gestão para carcinicultura (cultivo de camarão L. vannamei)

Firmware v9.40 — Junho 2026

O que é o AquaShrimpTech Bot?

Um sistema embarcado no ESP32-S3 que se comunica via Telegram para monitorar e controlar viveiros de camarão. Cada viveiro tem seu próprio ESP32-S3 com firmware dedicado. O técnico ou produtor opera tudo pelo celular, sem precisar estar presencialmente no viveiro.

O que o sistema faz

🌀 Controle de Aeradores

Liga/desliga aeradores remotamente via Telegram. Monitora corrente elétrica e alerta automaticamente se um aerador parar.

⚡ Monitoramento Elétrico

Mede tensão nas 3 fases (L1, L2, L3). Alerta em caso de fase zerada, sobretensão ou subtensão.

🦐 Gestão de Cultivo

Registra povoamento, biometria, ração diária, mortalidade, FCA, biomassa estimada e sobrevivência.

📦 Controle de Estoque

Gerencia entrada e saída de insumos (ração, produtos químicos, etc.) com alertas de estoque mínimo.

🧪 Dados Técnicos

Coleta temperatura, OD, pH, salinidade, transparência, amônia, nitrito e alcalinidade.

📊 Google Sheets

Todos os dados são enviados automaticamente para planilhas Google, acessíveis pelo painel web.

🔄 OTA (Update Remoto)

Firmware atualiza automaticamente via internet, sem precisar de cabo USB após a instalação inicial.

🔒 Tampa / Segurança

Sensor de fim de curso detecta abertura do painel elétrico e envia alerta imediato.

Arquitetura do Sistema

Fluxo de dados

ESP32-S3 (viveiro) — lê sensores a cada 5s, processa lógica local, persiste estado no NVS

Dashboard Web (GitHub Pages) — painel administrativo e do cliente, lê dados do Sheets em tempo real

Cada viveiro tem seu próprio ESP32-S3 e seu próprio Bot Telegram (token diferente). Todos os bots ficam no mesmo grupo Telegram do cliente. O chat_id é o mesmo para todos os ESPs de um cliente.

Seleção de Viveiro (como funciona)

O Telegram é um canal de broadcast — todos os bots do grupo recebem todas as mensagens. O mecanismo de seleção resolve quem age:

Usuário clica "🦐 VIVEIRO 02" — todos os ESPs recebem a mensagem
Cada ESP extrai o número e compara com o seu: ESP do V02 vê numClic==2 e numEste==2 → match! Salva grupoSelecionouEsteViveiro=true no NVS
ESP do V01, V03... veem que não é deles e salvam false. O guard bloqueia todos os comandos subsequentes neles.
Agora apenas o V02 responde a qualquer botão ou comando — até que outro viveiro seja selecionado

Provisionamento — Configuração Inicial

Provisionamento é o processo de gravar as credenciais do cliente no ESP32-S3 pela primeira vez. Feito uma única vez por device, via cabo USB + Serial Monitor.

Pré-requisitos

  • Arduino IDE instalado com suporte ao ESP32-S3
  • Driver USB instalado (CH340 ou CP2102 dependendo da placa)
  • Board selecionada: ESP32S3 Dev Module
  • Partition Scheme: Minimal SPIFFS (1.9MB APP with OTA/190KB SPIFFS)
  • Baud rate Serial Monitor: 115200
  • Line ending: Newline (Tools → Line ending → Newline)

Processo Passo a Passo

1
Criar o Bot Telegram

Abra o Telegram e fale com @BotFather. Digite /newbot, escolha um nome (ex: AquaShrimpTech V02) e um username (ex: ast_viveiro02_bot). Guarde o token gerado.

Um bot por viveiro. Nunca reutilize o mesmo token em dois ESPs.
2
Adicionar o Bot ao Grupo

No grupo Telegram do cliente, clique em "Adicionar membro" e procure pelo username do bot criado. Adicione-o como membro.

3
Obter o Chat ID do grupo

Adicione o bot @userinfobot ao grupo e envie qualquer mensagem. Ele responderá com o Chat ID do grupo (número negativo, ex: -1001234567890).

4
Gravar o firmware no ESP32-S3

Abra o arquivo viveiro_bot_v9_40.ino no Arduino IDE. Selecione a porta COM correta e clique em Upload.

5
Abrir o Serial Monitor e preencher os dados

Após o upload, abra o Serial Monitor (115200 baud, Newline). O ESP detecta que o NVS está vazio e inicia o provisionamento:

╔══════════════════════════════════════════╗
║     AQUASHRIMPTECH — PROVISIONAMENTO     ║
╚══════════════════════════════════════════╝

1/7 - SSID Wi-Fi do cliente       : MinhaRede2G
2/7 - Senha Wi-Fi                 : minhasenha123
3/7 - Token do Bot Telegram       : 7654321:AAFxxx...
4/7 - Chat ID do grupo            : -1001234567890
5/7 - URL Google Script viveiro   : https://script.google.com/macros/s/AKfycb.../exec
6/7 - URL Google Script estoque   : https://script.google.com/macros/s/AKfycb.../exec
7/7 - Nome do viveiro             : Viveiro 02

Digite cada valor e pressione Enter. Após o 7º campo, o ESP grava no NVS e reinicia automaticamente.

6
Verificar no Telegram

O bot enviará no grupo:

🦐 [Viveiro 02] online — v9.40
ℹ️ Selecione este viveiro no menu para operar

O provisionamento foi concluído com sucesso.

Reprovisionar (limpar e recomeçar)

Se precisar apagar tudo e começar do zero (erro, troca de cliente, etc.), use o sketch limpar_e_reprovisionar.ino:

  1. Grave o limpar_e_reprovisionar.ino no ESP
  2. Abra o Serial Monitor — ele apaga o NVS e pede os 7 campos novamente
  3. Após preencher, grave o firmware principal (viveiro_bot_v9_40.ino)
O NVS gravado pelo sketch de limpeza não é sobrescrito quando você grava o firmware principal — o firmware só provisiona se o NVS estiver vazio.

Alterar WiFi remotamente (sem cabo)

Se a rede WiFi do cliente mudar, use o comando no Telegram (com o viveiro selecionado):

/configurarwifi NovoNomeDaRede / NovaSenha123

O ESP salva no NVS e reinicia. Em 10 segundos está online na nova rede.

Estas constantes ficam no código e exigem recompilação + nova gravação quando alteradas:

🔧 Configuração de Sensores e Pinos

PINO_SCT[4]
{1,2,3,4}
GPIOs dos sensores de corrente SCT-013. Um por aerador. Use -1 para desabilitar um canal sem sensor físico.
PINO_ZMPT[3]
{5,6,7}
GPIOs dos sensores de tensão ZMPT101B. Um por fase elétrica (L1, L2, L3). Use -1 para desabilitar.
PINO_RELE[4]
{16,17,18,19}
GPIOs dos relés. Os 3 primeiros controlam aeradores. O 4º (GPIO 19) está reservado para expansão.
PINO_TAMPA
15
GPIO do sensor de fim de curso da tampa do painel. Configurado como INPUT_PULLUP. Não alterar sem hardware correspondente.
NUM_AERADORES
3
Quantos aeradores este viveiro tem. Máximo 4. Altera automaticamente os botões Ligar 1/2/3 e o monitoramento.

🔴 Define mais importante: SEM_SENSORES

#define SEM_SENSORES
true
true = sem sensores físicos instalados. Desabilita TODAS as leituras de corrente, tensão e tampa. Zero alertas automáticos. Use quando o viveiro ainda não tem hardware de medição.

false = sensores físicos instalados e funcionando. Habilita monitoramento completo, alertas automáticos e calibração.
✅ Para ativar os sensores: mude #define SEM_SENSORES true para #define SEM_SENSORES false, recompile e grave via OTA ou cabo.

🔵 Multi-viveiro no mesmo grupo

#define VIVEIRO_UNICO_NO_GRUPO
false
false = há múltiplos ESPs no mesmo grupo Telegram. O device só responde após ser selecionado pelo usuário. Use quando o cliente tem 2+ viveiros.

true = apenas 1 ESP no grupo. Responde a tudo sem precisar ser selecionado. Use apenas para cliente com 1 viveiro.

Comandos do Bot

Os comandos são enviados digitando no chat do grupo Telegram. Há dois tipos:

GLOBAL Todos os ESPs do grupo respondem simultaneamente
OPERACIONAL Só o viveiro selecionado responde

Comandos Globais (todos os ESPs respondem)

/versao
GLOBAL
Mostra versão do firmware, heap livre e status do NVS de cada ESP. Útil para confirmar que todos estão na mesma versão.
/showconfig
GLOBAL
Exibe o conteúdo do NVS: viveiro, SSID, chat_id, URLs. Use para verificar se o provisionamento foi correto. Token não aparece completo por segurança.
/ajuda
GLOBAL
Lista todos os comandos disponíveis com descrição resumida.
/limpafila
GLOBAL
Descarta mensagens acumuladas na fila do Telegram e reabre o socket SSL. Usar quando o bot parar de responder. Todos os ESPs executam — é intencional.

Comandos Operacionais (só viveiro selecionado)

Selecione o viveiro antes de usar estes comandos: clique em "🦐 VIVEIRO 0X" no picker.
/reiniciar
OPERACIONAL
Reinicia o ESP selecionado remotamente. Útil para testar restauração de relés sem ir ao local. O ESP volta sozinho em ~15 segundos.
/atualizar
OPERACIONAL
Força verificação de OTA imediatamente (sem aguardar as 24h). Baixa e instala nova versão se disponível no GitHub.
/simulacao
OPERACIONAL TOGGLE
Ativa/desativa modo simulação. Em simulação, tensões e correntes são geradas aleatoriamente (sem ler sensores reais). Útil para demo.
/recarregarestoque
OPERACIONAL
Recarrega a lista de insumos da planilha de estoque. Usar após adicionar novos insumos diretamente na planilha.
/calibragem
OPERACIONAL
Exibe os fatores de calibração atuais de cada fase (L1, L2, L3) e a tensão sendo lida com eles.
/calibrar <fase> <V_real>
OPERACIONAL
Calibra a tensão de uma fase usando um multímetro como referência. Exemplo: /calibrar 1 220 calibra L1 para 220V. Salvo no NVS — sobrevive a OTA.
/configurarwifi <ssid> / <senha>
OPERACIONAL
Atualiza WiFi remotamente sem cabo. Exemplo: /configurarwifi MinhaRede / senha123. O ESP salva e reinicia.
/configurarestoque <url>
OPERACIONAL
Define ou corrige a URL do Google Script do estoque compartilhado. Salvo no NVS.
/alterarsenha
OPERACIONAL
Altera a senha de acesso à planilha. Mínimo 4 caracteres.
/menu ou /start
ESPECIAL
Exibe o Painel Central de Viveiros (picker). Na versão atual, todos os ESPs respondem — será corrigido na v9.41.

Sensores — Instalação e Ativação

Estado atual: SEM_SENSORES

Quando #define SEM_SENSORES true, o ESP funciona normalmente para registro de dados (ração, biometria, custos, estoque), mas não lê nenhum sensor físico e não envia nenhum alerta automático de tensão, corrente ou tampa.

Para ATIVAR os sensores

1
Instale o hardware

Conecte os módulos SCT-013 nos GPIOs 1, 2, 3 (um por aerador) e os ZMPT101B nos GPIOs 5, 6, 7 (um por fase). Conecte o fim de curso da tampa no GPIO 15.

2
Mude o define no código 
// Linha ~74 do arquivo viveiro_bot_v9_40.ino
#define SEM_SENSORES false  // era true
3
Grave o firmware

Via cabo USB + Arduino IDE, ou via OTA se uma versão anterior já estiver rodando (atualize o arquivo no GitHub e dispare /atualizar).

4
Calibre a tensão (com multímetro)

Com o viveiro energizado e um multímetro na fase 1, verifique a tensão real. Se o bot mostrar valor diferente:

/calibrar 1 220.5

Repita para as fases 2 e 3. Os fatores ficam salvos no NVS — sobrevivem a OTA e reboot.

Solução de Problemas

🔴 Bot não responde a nada

Causa mais comum: fila do Telegram travada.

Solução: envie /limpafila no grupo. Todos os ESPs executam e limpam a fila.

Se não resolver: o ESP pode estar em loop de boot. Conecte via Serial Monitor e observe o log.

🔴 ESP em loop de boot (reinicia repetidamente)

Identificação: no Serial Monitor aparece "AQUASHRIMPTECH BOOT OK" repetidas vezes com intervalo de ~15 segundos.

Causas possíveis:

  • GPIO inválido: erro adc_io_to_channel(29): invalid gpio number → PINO_SCT[3] = 4 é inválido no S3. Mude para -1 ou ative SEM_SENSORES true.
  • Brownout: fonte USB fraca. Use carregador de celular 5V/1A+ em vez de porta USB do notebook.
  • Crash de código (PANIC): motivo do reset aparece no Serial como "PANIC". Envie o log.

🟡 Viveiro selecionado não responde aos botões

Causa: grupoSelecionouEsteViveiro = false no ESP — o guard bloqueando tudo.

Solução: clique novamente em "🦐 VIVEIRO 0X" no picker. A seleção é salva no NVS e sobrevive a reboot.

Se persistir: o ESP pode estar consumindo mensagens com delay. Aguarde 5 segundos após clicar.

🟡 Erro HTTP 409 no Serial

Causa: dois ESPs com o mesmo token tentando consumir a mesma fila do Telegram.

Solução: cada viveiro precisa de um token diferente. Crie novo bot no BotFather e reprovisione o ESP com token único.

🟡 Alertas falsos de tensão zerada (sem sensores)

Causa: SEM_SENSORES false com pinos físicos não conectados — ADC lê zero e dispara alerta.

Solução: mude para #define SEM_SENSORES true e regrave.

🟡 OTA falhou

Causas possíveis:

  • Partition Scheme errado: deve ser "Minimal SPIFFS (1.9MB APP with OTA)"
  • Arquivo .bin maior que 1.9MB
  • Conexão WiFi instável durante o download
  • Timeout de 30s sem dados

Solução: verifique o Partition Scheme no Arduino IDE e tente /atualizar novamente.

🔵 Watchdog Estágio 1/2 frequente

Estágio 1 (4min): o bot tenta auto-recuperar limpando a fila. Normal se houver instabilidade de WiFi.

Estágio 2 (7min): o ESP reinicia automaticamente. Se acontecer mais de 1x por dia, investigar sinal WiFi (RSSI aparece no heartbeat do painel ADM). RSSI abaixo de -70dBm é problemático — instalar antena externa.

🔵 Mensagem duplicada (menu aparece 2x)

Causa: /menu, /start e "Voltar ao Menu Principal" são globais — todos os ESPs respondem.

Status: comportamento cosmético, não quebra nada. Será corrigido na v9.41.

Como usar o Serial Monitor para diagnóstico

  1. Conecte o ESP via cabo USB
  2. Abra Arduino IDE → Tools → Serial Monitor
  3. Configure: 115200 baud
  4. Observe as linhas [LOG], [HEAP], [WD], [FILA] e [BOOT]
[BOOT] online v9.40 | reset: Power-on | heap 247KB   ← boot normal
[HEAP] free=250000  maxBloco=155000  min=128000       ← memória saudável
[WD] Estagio 1: tentando consumir (pending=1)          ← watchdog agindo
[LOG] TENSAO: Fase 1 (L1) ZEROU (0V)                  ← alerta de tensão
[FILA] Descartada. update_id=12345                     ← fila limpa

Log remoto no painel ADM

Todos os logs do Serial também são enviados ao painel administrativo via HTTP. Acesse o ADM e filtre por viveiro para ver histórico de eventos sem precisar de cabo USB.

  • ☐ Criar grupo Telegram com o cliente
  • ☐ Para cada viveiro: criar bot no BotFather e guardar token
  • ☐ Adicionar todos os bots ao grupo
  • ☐ Obter o Chat ID do grupo (via @userinfobot)
  • ☐ Para cada viveiro: copiar planilha template e publicar GAS → guardar URL
  • ☐ Criar planilha de estoque compartilhada → guardar URL

Parte 2 — Hardware e Firmware

  • ☐ Para cada ESP: gravar viveiro_bot_v9_40.ino via Arduino IDE
  • ☐ Verificar Partition Scheme: Minimal SPIFFS (1.9MB APP with OTA)
  • ☐ Se viveiro sem sensores ainda: confirmar #define SEM_SENSORES true
  • ☐ Se 2+ viveiros: confirmar #define VIVEIRO_UNICO_NO_GRUPO false
  • ☐ Para cada ESP: abrir Serial Monitor e preencher os 7 campos do provisionamento

Parte 3 — Verificação Final

  • ☐ Cada bot anuncia no grupo após boot
  • ☐ /showconfig em cada viveiro confirma dados corretos
  • ☐ /versao confirma firmware correto em todos
  • ☐ Picker funciona: clicar em cada viveiro e verificar resposta
  • ☐ Estoque carrega corretamente
  • ☐ Testar Informar Ração em pelo menos 1 viveiro
  • ☐ Se sensores instalados: verificar tensões no Status e calibrar se necessário
